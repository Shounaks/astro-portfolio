<Section link="iam-security-tools" title="Security Tools Provided by IAM" />
There are 2 tools provided by AWS to control the Identity and Access of users in
our organisation.
<ul class="list-disc pl-6">
	<li>IAM Credential Report (account level)</li>
	<li>IAM Access Advisor (user level)</li>
</ul>

<br />
<Section link="iam-credential-report" title="IAM Credential Report" />
It generates a report that lists all your account's user and the status of their
various credentials.
<Section link="iam-access-advisor" title="IAM Access Advisor" />
It shows the service permissions granted to a user and when those services were
last accessed. We can use this permission to revise your policy.

<Section link="iam-access-advisor" title="IAM best Practice" />
<ul class="list-disc pl-6">
	<li>Don't use root account except for access setup.</li>
	<li>One physical user should have only one AWS account.</li>
	<li>Assign users to group & Assign policy permissions to group.</li>
	<li>Create a strong password policy.</li>
	<li>Use & Enforce the use of MFA</li>
	<li>
		Create and Use roles to give one AWS service the ability to interact with
		another.
	</li>
	<li>Use Access Keys for Programmatic Access (CLI/SDK)</li>
	<li>
		Audit permission of your account using IAM Credential Report & IAM Access
		Advisor.
	</li>
	<li>NEVER SHARE YOUR IAM USER & ACCESS KEYS</li>
</ul>
