<Section
	link="aws-ecs"
	title="Elastic Container Service"
	imageUrl="/content/cloud/aws/icons/Arch_Amazon-Elastic-Container-Service_32.svg"
/>

- It elables you to launch Docker Containers on AWS (inside EC2 containers)
- <mark>You must provision and maintain the infrastructure!</mark>
- AWS takes care of starting and stopping the containers.
- Has integration with ALB (Application Load Balancer)
- New Docker Containers will be created in one of the connected EC2 instances. EC2 is smart enough toknow which one!

	title="AWS Fargete"

/>

- It is also used to launch Docker Images on AWS <mark>BUT!! you do not provision the infrastructure (i.e. NO EC2 Instance to manage)</mark>, hence simpler.
- It is a <mark>Serverless</mark> offering.
- AWS just runs containers for your requirements, based on teh CPU/RAM you need
