<Section link="aws-pricing" title="Pricing in AWS Cloud" />
AWS has 3 pricingfundamentals, based on the pay-as-you-go pricing model.
<ul class="list-disc pl-6">
	<li>Compute : Pay for Compute Time</li>
	<li>Storage : Pay for the amount of data Stored in cloud</li>
	<li>
		Data transfered out of the cloud. (NOTE: Data transferred inside is FREE)
	</li>
</ul>
<br />
<Section link="aws-region" title="How to  Select an AWS Region?" />
There are some questions we need to ask ourselves before selecting a region we want
our application to run in.

| Topic                 | Question                                          | Remarks                                                 |
| --------------------- | ------------------------------------------------- | ------------------------------------------------------- |
| Compliance            | What regulatory compliance does this region meet? | Data Governance/Legal Requirements                      |
| Proximity to Customer | What is the distance/latency to my end-user?      |                                                         |
| Available Services    | Is the service, available in the region?          | New AWS Services might not be available in some regions |
| Pricing               | What is the cost of AWS Service in this region?   | It varies from region to region & is transparent        |
