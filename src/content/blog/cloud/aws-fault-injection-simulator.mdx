- A fully managed service for running fault injection experiments.
- Based on Chaos Engineering - Stressing an application by creating disruptive events (e.g. Sudden Increase in CPU or Memory),
    then observing how system respponds to it and implementing improvements.
- Helps you uncover hidden bugs and peformance bottlenecks.
- Supports: EC2, EKS, ECS, RDS, etc
- Use pre-built templates that generate the desired disruption.
